# Daily Growth Planner

Локальное React-приложение для ежедневного развития видеографа: работа, обучение, здоровье, набор массы, личный бренд, портфолио и контроль срывов. Интерфейс полностью на русском языке.

## Что работает

- персональный план дня по энергии, рабочим часам, спорту и риску алкоголя;
- расписание, питание для набора массы и мягкий режим восстановления;
- 100 учебных тем и недельная программа «Middle → Strong»;
- Skill Tree из семи направлений с практиками и датами повторений;
- трекеры веса, тренировок, питания, воды, сна, курения и энергии;
- генератор из 27 сценариев Reels/сторис в девяти категориях;
- портфолио с семью типами проектов и отметками для шоурила;
- шаблоны нетворкинга и недельный план `2 клиента + 2 режиссёра + 1 follow-up`;
- анти-срыв протокол и подробный вечерний разбор;
- календарь прогресса по пяти ежедневным минимумам;
- экспорт плана, недели, Telegram-текста, `.txt` и полной JSON-копии;
- импорт и перенос полной JSON-копии между Mac и iPhone;
- вход по email и автоматическая синхронизация через Supabase;
- offline-очередь, ручная синхронизация и безопасное разрешение конфликтов;
- локальные напоминания с настраиваемым временем и днями;
- светлая и тёмная темы, адаптивная навигация для iPhone и ноутбука.
- установка как PWA на iPhone и Mac;
- offline-режим после первого успешного открытия приложения.

## Запуск

Требуется Node.js 18 или новее.

```bash
npm install
npm run dev
```

Vite покажет локальный адрес, обычно `http://localhost:5173`.

Опубликованная версия: [Daily Growth Planner](https://abbosroziyevww-jpg.github.io/daily-growth-planner/).

## Production-сборка

```bash
npm run build
npm run preview
```

Команда `npm run build` сначала запускает строгую проверку TypeScript, затем создаёт оптимизированную сборку в `dist/`.

Для установки PWA production-сайт должен работать по HTTPS. Исключение — локальная разработка на `localhost`.

## Как установить на iPhone

1. Открыть сайт в Safari.
2. Нажать кнопку **Share** («Поделиться»).
3. Нажать **Add to Home Screen** («На экран Домой»).
4. Открыть Daily Growth Planner с главного экрана.

Приложение запускается без интерфейса Safari, использует собственную иконку и splash screen. После первого открытия основные экраны доступны без сети.

## Как установить на Mac

1. Открыть сайт в Chrome или Safari.
2. В Chrome нажать **Install App** в адресной строке; в Safari выбрать **File → Add to Dock**.
3. Запустить Daily Growth Planner как отдельное приложение из Dock, Launchpad или папки Applications.

## PWA и offline-режим

- manifest: `manifest.webmanifest`;
- service worker создаётся автоматически при `npm run build`;
- обновления service worker применяются автоматически;
- интерфейс, стили, иконки и основные данные кэшируются для offline-работы;
- пользовательские записи по-прежнему хранятся в LocalStorage и не пропадают при обновлении PWA.

## Хранение, синхронизация и резервная копия

Все данные сначала автоматически сохраняются в LocalStorage текущего браузера под ключом `daily-growth-planner-v1`. Поэтому приложение продолжает работать без интернета. После входа изменения также отправляются в Supabase; при восстановлении сети синхронизация запускается автоматически.

JSON-экспорт и импорт остаются дополнительной защитой и не включают пароль, токен входа или служебные данные синхронизации.

Для переноса откройте раздел «Данные и резервная копия»: скачайте файл `abbos-growth-planner-backup-YYYY-MM-DD.json`, передайте его через AirDrop, iCloud Drive, Files или Telegram и импортируйте на втором устройстве. Импорт поддерживает также старые JSON-файлы из раздела «Экспорт».

Старые данные приложения автоматически дополняются новыми полями при загрузке.

## Настройка Supabase

1. Создайте проект на [Supabase](https://supabase.com/dashboard).
2. Откройте **SQL Editor**, вставьте содержимое файла `supabase/schema.sql` и выполните запрос. Он создаст таблицу `user_data` и политики RLS: пользователь видит только собственную строку.
3. В **Authentication → Providers → Email** оставьте включённым вход по email. При включённом подтверждении email пользователь должен открыть письмо перед первым входом.
4. В **Project Settings → API** скопируйте Project URL и publishable key (или legacy `anon` key). Никогда не используйте `service_role` в браузере.
5. Скопируйте `.env.example` в `.env.local` и заполните значения:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_PUBLISHABLE_OR_ANON_KEY
```

6. Перезапустите `npm run dev`, откройте раздел «Аккаунт и синхронизация» и зарегистрируйтесь. На Mac и iPhone войдите с одним email и паролем.

Для email-подтверждения опубликованной версии добавьте адрес сайта в **Authentication → URL Configuration → Site URL** и **Redirect URLs**.

### GitHub Pages

В репозитории откройте **Settings → Secrets and variables → Actions** и добавьте два repository secret:

- `VITE_SUPABASE_URL`;
- `VITE_SUPABASE_ANON_KEY`.

Workflow `.github/workflows/deploy-pages.yml` передаст их только на этапе production-сборки. После добавления секретов запустите workflow заново или отправьте новый commit.

### Правила синхронизации

- любое изменение немедленно сохраняется локально;
- облако обновляется после короткой паузы, при открытии приложения, возвращении сети и периодически во время работы;
- если обе версии изменились на разных устройствах, приложение ничего не перезаписывает и предлагает выбрать облачную или локальную версию;
- кнопка «Синхронизировать сейчас» запускает проверку вручную;
- резервные JSON-файлы продолжают работать независимо от Supabase.

## Основная структура

```text
src/
├── components/   интерфейсные модули и трекеры
├── data/         темы обучения, Skill Tree, идеи и шаблоны
├── utils/        генераторы плана дня и учебной задачи
├── App.tsx       навигация, состояние и LocalStorage
├── types.ts      общие TypeScript-типы
└── index.css     Tailwind и визуальная система
```

## Адаптивность

- нижняя навигация и safe-area для современных iPhone;
- полный список разделов в мобильном меню «Ещё»;
- карточки и таблицы перестраиваются в одну колонку;
- широкие графики и фильтры получают горизонтальную прокрутку;
- на экранах `lg` и шире используется постоянная боковая панель.

## Ограничения текущей версии

- Supabase нужно один раз настроить и добавить переменные окружения;
- напоминания показываются внутри открытого приложения, это не системные push-уведомления;
- калории и медицинские показатели являются ориентирами, а не медицинской рекомендацией.

## Идеи для следующих версий

- системные push-уведомления;
- редактирование пользовательской базы учебных тем;
- расширенная аналитика по неделям и месяцам;
- интеграция с календарём, Telegram-ботом и Apple Health.
